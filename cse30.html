Purpose:

Practice manipulating C strings by writing a program to process a csv-formatted file. See each file for a more specific description per file.

Execution:

We were only allowed to use pointers and no indexing. Taking in input from the command line, we convert each argument to an integer and stored in an array. Then, using the arguments, we split the buffer to get only certain rows or columns. Finally, we write our csv rows to output.

Purpose:

Work more with C strings and pointers. Work with hash tables, linked lists, memory allocation, and structs. Pull it all together to create a parking database in which "tickets" for certain offenses are stored for certain vehicles, and users can view or pay their tickets.

Execution:

We weren't allowed to use indexing, so everything had to be pointers. There was a hashtable of vehicles, and each vehicle had a chain of tickets. We used structs for vehicles, tickets, and fines.

Purpose:

Create a DFA that would process text like a C preprocessor. Then, use that DFA and its states and state transitions to perform preprocessor functions for comments (block and line) and strings.

Execution:

We drew out the DFA, considering states and their transitions. For example, if the current char was a quote, we would either enter or exit a 'string' state, depending on our current state. Then we implemented each state and a main and tested our code with many different strings and/or comments.

Purpose:

Work with bit shifting to encrypt and decrypt messages. Read files and use fwrite(). Manually set up the stack frame in assembly.

Execution:

Decrypt with a BOOK file, process a buffer and encrypt. Write a main to take in stdin, the buffer, and the bookfile. Then, write all these again in assembly. This project was definitely the most difficult because the assembly was difficult to debug, but going through step by step and checking register values with GDB helped. Plus, the stack frame required adding a 5th argument so we had to make sure to put that in the right place.
